General questions:

What is a "gazetteer"?

Most people may think of "gazette," as in a newspaper, but a gazetteer is actually completely different. As early as Ancient Greece, gazetteers were publications that contained geographic and demographic information about a location. Gazetteer HK is a modern day version of this concept, collecting a variety of data sources into a single unified and standardized database, available for anyone to use as a basis for their use case.

See the Wikipedia entry in English and Chinese for more history about gazetteers.

What kind of use cases are there for Gazetter HK?

How much does it cost to use the API?

Nothing! At the moment, we don't have plans to charge for API usage. Our project is also fully open source, and contains the scripts necessary to generate our full dataset.

How do I use this?

If you're interested in seeing visualizations of the data, check out some of our pre-made apps, such as the Neighborhood Profilesa>, or the Data Mapper.

If you're interested in using the API, check out the Data Browser for documentation, and visit our GitHub page for the source code.

Can I help contribute?

Absolutely. Check out some of the issues on our issue tracker, fork the code on Github and send a pull request. If you have any questions, get in touch with us at contact@gazetteer.hk.

What's your stack?

Our backend API was built with Python using Flask. Flask interacts directly with a pandas DataFrame to query the data, so there is no database involved.

Our frontend is built primarily using AngularJS, Leaflet, and dimple, which builds on top of the D3js framework. For translations, we use i18next. There are also a handful of other plugins that we use. Check out our bower file for full details.
